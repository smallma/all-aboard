  }

  - 可寫 Python/Node 輔助腳本放 scripts/measure-seats.py
ACCEPTANCE:
  - [ ] docs/agents/seat-coordinates.json 存在且 JSON 格式正確
  - [ ] 四種車型都有 driver / seats / trunk 三個欄位
  - [ ] 座位數量符合規格：轎車=4、MPV=6、貨車=1、皮卡=1
  - [ ] 所有 xPct/yPct/wPct/hPct 為 0~100 整數
  - [ ] xPct 是水平百分比（0=左、100=右）
  - [ ] yPct 是垂直百分比（0=上、100=下）
---
規格：

每張車型需提供：
- driver: 駕駛座中心點 { xPct, yPct }
  - 駕駛座在所有車型都是「左上區域」（方向盤所在位置）
- seats[]: 副駕 + 後座每個位子的中心點
  - label 用中文：副駕 / 中排-左 / 中排-右 / 後排-左 / 後排-中 / 後排-右
  - 轎車 seats: 副駕(0) + 後排-左(1) + 後排-中(2) + 後排-右(3)
  - MPV seats: 副駕(0) + 中排-左(1) + 中排-右(2) + 後排-左(3) + 後排-中(4) + 後排-右(5)
  - 貨車 seats: 副駕(0)
  - 皮卡 seats: 副駕(0)
- trunk: 後車廂矩形 { xPct, yPct, wPct, hPct }
  - xPct/yPct 是矩形中心
  - wPct/hPct 是矩形寬高百分比
  - 貨車的 trunk 區域最大（HUGE CARGO SPACE）
  - MPV 的 trunk 較小

建議做法（不強制）：
- 寫 scripts/measure-seats.py 用 PIL 讀 PNG
- 你可以手動標註關鍵點，腳本算百分比
- 或直接用圖工具看像素位置硬算

注意：
- 不必精準到小數第二位，整數即可
- 大概對齊就好，claude 之後會在 UI 上微調
- 座位中心點略偏向「人會坐進去」的位置（座椅靠背中心）

注意：
- 不要做 canDrive 驗證！那是 store 層 dragdrop action 的事
- 圖示用 lucide-react 的 Steering（或 SteeringWheel；若無就用 Car icon）
- 元件不接受 children；單純呈現+droppable

範例骨架（你可調整）：

import { useDroppable } from '@dnd-kit/core';
import { Steering } from 'lucide-react';  // 若 lucide 沒有，改用 Car
import { clsx } from 'clsx';
import { buildDndId } from '@/lib/dnd-id';

export function DriverSeat({ carId, driverId, xPct, yPct }: Props) {
  const { setNodeRef, isOver } = useDroppable({
    id: buildDndId({ kind: 'driver', carId }),
  });

  return (
    <div
      ref={setNodeRef}
      style={{ left: `${xPct}%`, top: `${yPct}%` }}
      className={clsx(
        'absolute -translate-x-1/2 -translate-y-1/2',
        'w-12 h-12 rounded-full flex items-center justify-center',
        driverId ? 'bg-amber-100' : 'border-2 border-dashed border-slate-300',
        isOver && 'ring-2 ring-blue-400',
      )}
    >
      {driverId ? <span className="text-xs">已坐</span> : <Steering className="w-5 h-5 text-slate-400" />}
    </div>
  );
}
